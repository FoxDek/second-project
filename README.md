# Основы работы с Git: определения и команды

---

VCS (Version Control System) - программа, которая хранит историю IT-проектов

Одна из ключевых особенностей современных систем контроля версий - поддержка параллельной работы нескольких пользователей, в том числе над одним файлом => VCS популярны в IT

VCS - общее название ряда продуктов, таких как Git, Mercurial, Subversion и др. (Git - самый популярный из них)

Git используется для:
1. Синхронизации изменений, сделанных разными участниками команды
2. Хранения истории изменений файлов при командной работе
3. Хранения нескольких версий проекта, над котороым работает один человек

---

## Основные команды и обозначения

Символ доллара (**$**) означает, что программа ждёт команд

**pwd** (print working directory) - покажет путь к текущей директории
=> ком. стр. (командная строка) выведеть путь к домашней директории - каталогу с файлами пользователя

**cd ~** (change directory + '~' - домашняя директория) - переход к домашней директории

**ls** (list directory contents) - отобразить содержимое директории

**cd** (change directory) - меняет текущую рабочую директорию на ту, которая указана в качестве параметра (т.е. после команды)

Чтобы вернуться обратно (на уровень выше) вместо названия папки нужно написать две точки (**$ cd ..**)

Чтобы запустить скрипт* / программу используют одну точку (**$ cd .**)

* - скрипт - небольшая программа для выполнения конкретной задачи

---

У многих команд есть дополнительные опции

$ **ls** (вывести список файлов)

$ **ls -a** (список скрытых файлов, т.е. тех, которые начинаются с символа '.')

---

**touch** - команда создания файла, которая использует имя файла как параметр

**mkdir** (make directory) - команда для создания директорий (т.е. папок)

Можно создать целую структуру с помощью флага (**-p**)

_По умолчанию **touch** и **mkdir** создают файлы и папки в текущей директории, но можно создавать и в удалённых_

**cp** (copy) - команда для копирования файлов. В простом виде принимает всего 2 параметра (что копируем и куда копируем)

**mv** (move) - переместить файл/папку (синтаксис аналогичен с **cp**)

**cat** (concatenate and print) - команда для чтения файла, которая используется с именем файла

**rm** (remove) - команда для удаления файлов, которая использует имя файла как параметр

**rmdir** (remove directory) - команда для удаления папки

_Если в удаляемой папке есть файлы, то ком. стр. не удалит её и выведет сообщение о том, что папка не пуста. Это защита от случайного удаления_

**rm -r** (recursive) - команда рекурсивно удаляет файлы и папки. Это значит, что удаление будет последовательно применяться к каждому файлу, пока не сотрёт их все.

_Удаление объектов командами **rm** и **rmdir** необратимо_
